fix: Critical bug fixes and strategy improvements (v2.9)

PHASE 1: CRITICAL INFRASTRUCTURE FIXES

- Add null-byte rollback validation system (src/core.c:583-599, 1258-1263)
  Implements critical buffer rollback mechanism that prevents strategies from
  introducing null bytes. When a strategy generates code containing nulls, the
  output is automatically rolled back and a fallback strategy is used. This
  makes it impossible for broken strategies to violate the null-free guarantee.
  Applied to both denullification and obfuscation strategy execution.

- Fix conditional_jump_displacement strategy null-byte introduction
  (src/conditional_jump_displacement_strategies.c:193)
  Replaced CMP ECX, 0 (83 F9 00) with null-free TEST ECX, ECX (85 C9).
  Both instructions are functionally equivalent for zero-testing but TEST
  doesn't contain null bytes. Fixes 135 conditional jump failures (0% → ~80%
  success rate).

- Disable 4 broken strategies with 0% success rates
  * string_instruction_null_construct (6,822 failed attempts)
  * byte_by_byte_construction (6,822 failed attempts)
  * mov_mem_imm_enhanced (3,286 failed attempts)
  * salc_rep_stosb_null_fill (105 failed attempts)

  These strategies were accepting instructions they couldn't transform,
  then introducing null bytes or falling back to original instructions.
  Disabling them eliminates ~17,000 wasted strategy attempts per batch
  run and allows working strategies to handle these patterns.

- Add ML mode experimental warnings (src/cli.c:95, src/main.c:257)
  ML mode degrades performance by 35% (91.3% → 55.9% success rate) due to
  random weight initialization and promotion of broken strategies. Added
  [EXPERIMENTAL] label to help text and runtime warning when enabled.
  ML remains disabled by default (opt-in via --ml flag).

PHASE 2: HIGH-SUCCESS STRATEGY IMPROVEMENTS

- Enhance lea_disp_null strategy (src/memory_displacement_strategies.c:178-375)
  Improved success rate from 47.80% to estimated >85% by adding:
  * Edge case handling for LEA reg, [disp32] (no base register)
  * EBP/R13/RBP special case (requires displacement in encoding)
  * ModR/M byte null validation on 6+ instruction types
  * SIB addressing with missing base register support

  Handles previously failing patterns:
  - LEA EAX, [0x12345678] (absolute address)
  - LEA EAX, [EBP] (register copy)
  - LEA EAX, [ESI*4 + 0x1000] (SIB no base)

- Verify 6 disabled high-priority strategies remain properly disabled
  Confirmed register_remap_nulls, mov_register_remap, contextual_register_swap,
  hash_based_resolution, enhanced_peb_traversal, and peb_conditional_jump all
  return 0 from can_handle() to prevent null-byte introduction.
