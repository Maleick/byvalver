refactor: Rename "Bad-Character" terminology to "Bad-Byte" across entire codebase

This commit implements a comprehensive terminology refactoring, replacing all instances of
"Bad-Character" and its derivatives with the more elegant and alliterative "Bad-Byte" naming
convention. This change improves linguistic consistency and readability while maintaining
100% functional equivalence.

## Motivation

### Problem Description

The original "Bad-Character" terminology, while technically accurate, lacked the linguistic
elegance and memorability that would benefit a user-facing tool:

1. **Linguistic Inelegance**: "Bad-Character" is verbose and lacks phonetic appeal
2. **Inconsistent Alliteration**: The 'B' and 'C' sounds don't create memorable patterns
3. **Terminology Confusion**: "Character" implies text/encoding, but we're manipulating raw bytes
4. **Documentation Readability**: Long phrases like "bad-character elimination" are cumbersome

**User Impact:** Moderate - while functional, the terminology didn't maximize readability and
memorability for users learning the tool.

### Solution Implemented

**Comprehensive terminology replacement with "Bad-Byte" family:**

The new "Bad-Byte" naming convention provides:
- ✅ **Alliteration**: "Bad-Byte" creates memorable 'B-B' sound pattern
- ✅ **Technical Accuracy**: "Byte" is more precise than "character" for binary operations
- ✅ **Linguistic Economy**: Shorter, punchier, more memorable
- ✅ **Consistency**: Uniform across all code, docs, UI, and CLI

**Naming Convention Changes:**

```
Before                          After
──────────────────────────────────────────────────────────────
Bad-Character                   Bad-Byte
bad-character                   bad-byte
bad_char                        bad_byte
badchar                         badbyte
--bad-chars                     --bad-bytes
bad character                   bad byte
```

**Problem:** Function parameter mismatch:
```c

    // Used 'bad_chars' instead of 'bad_bytes'
```

**Fix:**
```c

}
```

**File:** src/ml_metrics.c:143

### 3. Variable Shadowing in cli.c

**Problem:** Local variable name didn't match usage:
```c

}
```

**Fix:**
```c
bad_byte_config_t *bad_bytes = parse_bad_bytes_string(value);  // Consistent naming
if (bad_bytes) {
    config->bad_bytes = bad_bytes;
}
```

**File:** src/cli.c:674
