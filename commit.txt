FEATURE: Add ML training utility and enhance path resolution for model loading

## Summary

This commit adds a complete ML training pipeline to BYVALVER with a standalone training utility and enhanced model path resolution. The training utility enables users to train custom ML models on their own shellcode datasets, while enhanced path resolution ensures the application works correctly regardless of where it's located.

## Problem

Original BYVALVER had ML functionality but:
1. No standalone training utility to train custom models
2. ML model paths were hardcoded and didn't work when executable was moved
3. No easy way to retrain the ML model on custom datasets
4. Missing documentation for ML training capabilities

## Solution

### 1. ML Training Utility
Added a standalone `train_model.c` utility that can train the ML model on custom shellcode datasets.

### 2. Enhanced Path Resolution
Updated both main executable and training pipeline to use dynamic path resolution:
- Uses `readlink` on `/proc/self/exe` to determine executable location
- Constructs model path relative to executable location
- Includes fallback to default path if resolution fails
- Works correctly regardless of executable location

### 3. Makefile Integration
Added `train` target to build the standalone training utility:
- Excludes `main.c` to avoid duplicate main function
- Includes all other source files for ML functionality
- Links properly with required libraries

### 4. Documentation Updates
Updated README.md, USAGE.md, and BUILD.md to document ML training capabilities.

## Technical Implementation

### New Implementation Files
- `src/train_model.c` - Standalone training utility
- Enhanced path resolution in `src/main.c` and `src/training_pipeline.c`

### Key Technical Approaches Implemented

#### 1. Dynamic Path Resolution
```c
char exe_path[PATH_MAX];
ssize_t len = readlink("/proc/self/exe", exe_path, sizeof(exe_path) - 1);
if (len != -1) {
    exe_path[len] = '\0';
    // Extract directory from executable path
    char *last_slash = strrchr(exe_path, '/');
    if (last_slash) {
        *last_slash = '\0';
        // Safely construct the model path
        size_t exe_len = strlen(exe_path);
        size_t suffix_len = strlen("/../ml_models/byvalver_ml_model.bin");
        if (exe_len + suffix_len < sizeof(model_path)) {
            strcpy(model_path, exe_path);
            strcat(model_path, "/../ml_models/byvalver_ml_model.bin");
        } else {
            // Fallback if path would be too long
            strncpy(model_path, "./ml_models/byvalver_ml_model.bin", 
                    sizeof(model_path) - 1);
        }
    }
}
```

#### 2. Training Utility Build Target
```makefile
TRAIN_TARGET = train_model
$(BIN_DIR)/$(TRAIN_TARGET): $(BIN_DIR) $(OBJS) decoder.h
	@echo "[LD] Linking $(TRAIN_TARGET)..."
	@$(CC) $(CFLAGS) -o $@ $(SRC_DIR)/train_model.c 
	$(filter-out $(SRC_DIR)/main.c, $(SRCS)) $(LDFLAGS)
	@echo "[OK] Built $(TRAIN_TARGET) successfully"

train: $(BIN_DIR)/$(TRAIN_TARGET)
```

### Updated Build Process
- Default build: `make` (no change)
- Training utility: `make train` (new target)
- Both include proper feature test macros for PATH_MAX
