infra: Add project infrastructure, CI/CD, Docker, docs hierarchy, and community files

Comprehensive open-source infrastructure overhaul to improve developer
onboarding, project reliability, and community contribution pathways.

Changes:
========

1. Documentation Hierarchy (docs/)
   - Created docs/README.md as central documentation index
   - Copied 10 docs from assets/docs/ into root-level docs/ for GitHub rendering
   - USAGE.md, BUILD.md, TUI_README.md, DENULL_STRATS.md, OBFUSCATION_STRATS.md
   - BAD_BYTE_PROFILES.md, BADBYTEELIM_STRATS.md, STRATEGY_HIERARCHY.md
   - ADVANCED_STRATEGIES.md, WHITEPAPER.md

2. README.md Restructure
   - Added hierarchical Table of Contents (35 entries with subsection nesting)
   - Added architecture maturity table (Stable v4.2 / Experimental v0.1)
   - Added ML maturity indicator (Beta v2.0)
   - Added Documentation section with table linking all docs/ files
   - Updated Development section with links to tests, Docker, contributing, roadmap

3. CI/CD Pipeline (.github/workflows/ci.yml)
   - GitHub Actions workflow triggered on push to main and PRs
   - Ubuntu latest with apt-based dependency installation
   - Build verification via make, test execution via tests/run_tests.sh

4. Docker Support
   - Dockerfile: Ubuntu 22.04 base with gcc, nasm, capstone, ncurses, python3
   - docker-compose.yml: Dev build service with volume mounts for source and output

5. Testing Infrastructure (tests/)
   - tests/run_tests.sh: Master test runner (build, fixture processing, verification)
   - tests/README.md: Test documentation and execution guide
   - tests/fixtures/{x86,x64,arm}/: Architecture-specific test fixture directories

6. Community Governance
   - CONTRIBUTING.md: Bug reporting, strategy addition, code style, PR process
   - CODE_OF_CONDUCT.md: Contributor Covenant v2.1 reference
   - ROADMAP.md: Feature maturity levels, v4.3/v5.0/v6.0 milestones

7. GitHub Templates
   - .github/ISSUE_TEMPLATE/bug_report.md: Structured bug report form
   - .github/ISSUE_TEMPLATE/feature_request.md: Feature proposal template
   - .github/PULL_REQUEST_TEMPLATE.md: PR checklist with testing section

8. Developer Configuration
   - .clang-format: LLVM-based C style config (4-space indent, Linux braces)
   - requirements-dev.txt: Python 3.6+ requirement documentation

Rationale:
==========

A mature C project with 212 source files and 170+ strategies lacked
standard open-source infrastructure: no CI/CD, no Docker, no contribution
guidelines, no roadmap, and documentation buried in assets/docs/ instead
of the GitHub-standard docs/ directory. These additions:

- Enable zero-to-working in under 5 minutes via Docker or quickstart
- Provide automated build/test verification on every push and PR
- Lower the barrier to contribution with clear guidelines and templates
- Signal feature maturity explicitly (Stable vs Experimental)
- Organize documentation for discoverability on GitHub
