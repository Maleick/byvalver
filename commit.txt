feat(ml): Fix training pipeline and add configurable training directory

## Summary

Fixed critical bugs in the ML training pipeline and made the train_model utility
production-ready by adding command-line configuration and comprehensive documentation.
Training now successfully processes shellcode samples and generates usable ML models.

## Key Fixes

### 1. Fixed Strategy Registry Initialization (Critical Bug)

**Problem**: Training collected 0 samples because strategy registry was never
initialized, causing `get_strategies_for_instruction()` to return 0 strategies.

**Solution**: Added `init_strategies(0)` call in `training_pipeline_execute()`
before data generation phase.

**Files Changed**:
- src/training_pipeline.c:
  - Added strategy.h include
  - Added init_strategies() call at line 364
  - Added initialization logging

**Impact**: Training now successfully identifies applicable strategies and collects
training samples. Tested with 558 .bin files, collected 10,000 samples (~31 MB).

### 2. Made Training Directory Configurable (Enhancement)

**Problem**: Training directory was hardcoded to machine-specific path
`~/RUBBISH/BIG_BIN`, making the tool unusable on other systems.

**Solution**: Converted train_model to accept required command-line argument for
training directory path.

**Files Changed**:
- src/train_model.c:
  - Added command-line argument parsing (argc, argv)
  - Added print_train_usage() function with help message
  - Added -h/--help flag support
  - Added validation for required directory argument
  - Added error handling for invalid options

- src/training_pipeline.c:
  - Removed hardcoded ~/RUBBISH/BIG_BIN fallback
  - Changed default to empty string (forces user to specify)
  - Training directory must now be provided as argument

**Usage**:
```bash
# Show help
./bin/train_model --help

# Train with custom directory (required)
./bin/train_model /path/to/shellcodes
./bin/train_model ~/my_training_data
./bin/train_model ./training_bins
```

**Impact**: Tool is now portable and usable on any system with shellcode samples.

### 3. Comprehensive Documentation (Enhancement)

**Problem**: ML_INTEGRATION.md had minimal documentation about train_model utility
and showed outdated examples without required arguments.

**Solution**: Added complete "The train_model Utility" section with step-by-step
guide, troubleshooting, and accurate examples.

**Files Changed**:
- docs/ML_INTEGRATION.md:
  - Added 200+ lines of train_model documentation (lines 154-355)
  - Documented 6-step training pipeline:
    1. Strategy Registry Initialization
    2. Training Data Generation
    3. Model Training (50 epochs, gradient descent)
    4. Model Evaluation (validation metrics)
    5. Model Persistence (saves to ml_models/)
    6. Performance Summary (metrics export)
  - Added training data requirements and sources
  - Added complete example training session with realistic output
  - Added troubleshooting section with common issues
  - Updated all examples to show required directory argument
  - Documented default training configuration parameters

**Impact**: Users now have complete reference documentation for training ML models.

## Breaking Changes

⚠️ **COMMAND-LINE CHANGE**: train_model now requires directory argument

**Before**:
```bash
./bin/train_model  # Used hardcoded ~/RUBBISH/BIG_BIN
```

**After**:
```bash
./bin/train_model /path/to/training/data  # Required argument
```

**Migration**: Simply provide the path to your training data directory when
running train_model. No code changes needed.
