feat: Enhance byvalver with comprehensive global CLI functionality

This commit implements comprehensive global CLI functionality for byvalver, transforming it from a locally-built tool to a professional-grade command-line application that can be installed system-wide across multiple platforms.

## Core Changes

### CLI Infrastructure
- Implemented proper argument parsing using POSIX `getopt` library
- Added comprehensive CLI module with `cli.h` and `cli.c`
- Created structured configuration system with default values
- Implemented proper error handling with standardized exit codes

### Command-Line Options
- Added general options: `--help`, `--version`, `--verbose`, `--quiet`, `--config`, `--no-color`
- Added processing options: `--biphasic`, `--pic`, `--xor-encode`, `--format`
- Added advanced options: `--strategy-limit`, `--max-size`, `--timeout`, `--dry-run`, `--stats`
- Added output options: `--output`, `--validate`

### Installation System
- Added `install` and `uninstall` targets to Makefile
- Created installation script `install.sh` for end users
- Added man page generation and installation
- Created example configuration file `.byvalverrc.example`

### Documentation Updates
- Updated README.md with new installation instructions and usage examples
- Created comprehensive USAGE.md documentation
- Updated BUILD.md with global installation instructions
- Added detailed option descriptions and examples

### Code Architecture
- Modularized CLI functionality in separate files
- Maintained backward compatibility with existing functionality
- Added proper configuration management system
- Implemented clean separation between core functionality and CLI interface

## Key Features

1. **Global Installation**: Install to `/usr/local/bin` with `sudo make install`
2. **Comprehensive Help**: Detailed help system with `--help` and man page
3. **Configuration Support**: JSON configuration file support with `--config`
4. **Multiple Output Formats**: Support for raw, C, Python, PowerShell, and hexstring formats
5. **Advanced Processing**: Biphasic processing, PIC generation, XOR encoding
6. **Validation**: Input validation with `--dry-run` and output validation
7. **Performance Controls**: Size limits, timeouts, and strategy limits

## Usage Examples

```bash
# Global installation
sudo make install
sudo make install-man

# Basic usage
byvalver input.bin output.bin

# Advanced usage
byvalver --biphasic --xor-encode 0x12345678 --verbose input.bin output.bin

# Validation
byvalver --dry-run input.bin

# With configuration file
byvalver --config myconfig.json input.bin output.bin
```

This implementation provides a professional-grade CLI experience with proper error handling, comprehensive documentation, and flexible configuration options.
