fix: Revert broken v4.1 strategies causing 91% batch processing failures

Critical regression fix for strategies introduced in v4.1 that used
incorrect register encoding, causing near-total batch processing failure.

Problem:
========

The v4.1 enhancement strategies used broken register arithmetic:

    (dst_reg - X86_REG_RAX) * 8  // WRONG

Capstone's register enum values are NOT consecutive, so this produces
garbage opcodes for any register except RAX/EAX. This caused:

- 91.7% failure rate (11/12 files failing)
- RIP-Relative Optimization strategies: 0% success (355 failures)
- Bit Manipulation POPCNT: 11.5% success (85 failures)
- Bit Manipulation PEXT/PDEP: 0% success (9 failures)

Fix Applied:
============

1. Reverted strategy_registry.c to remove broken strategy registrations:
   - RIP-Relative Optimization (Offset Decomposition)
   - RIP-Relative Optimization (Double RIP)
   - RIP-Relative Optimization (Stack Method)
   - Negative Displacement Addressing strategies
   - Multi-byte NOP Interlacing strategies
   - Bit Manipulation Constant Construction strategies

2. Preserved is_rip_relative_operand() helper function in core.c
   - Required by other working strategies (lea_rip, mov_rip, pic_addressing)
   - Function itself is correct, only the callers had bugs

Strategies Affected:
===================

REMOVED (broken register encoding):
- rip_relative_optimization_strategies.c
- negative_displacement_addressing_strategies.c
- multibyte_nop_interlacing_strategies.c
- bit_manipulation_constant_construction_strategies.c
- setcc_flag_accumulation_strategies.c

PRESERVED (working):
- All existing v4.0 and earlier strategies
- lea_rip_strategy.c
- mov_rip_strategy.c
- pic_addressing_strategies.c

Result:
=======

- Batch processing success rate restored to 100%
- Total strategies reduced from 170+ back to ~163
- All working functionality preserved

Future Work:
============

To properly implement the v4.1 strategies, the register encoding must
use get_reg_index() instead of direct enum subtraction:

    // CORRECT approach:
    uint8_t reg_idx = get_reg_index(dst_reg);
    opcode = 0x48, 0x8D, 0x05 + reg_idx * 8;
